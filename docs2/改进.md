# 提示词系统复核与改进建议

## 一、复核目标

本次复核的目标是先不动代码，先把现有提示词体系梳理清楚，确认：

1. 提示词是否已经按职责分层
2. 视频 / 生图提示词是否存在重复字段、无关字段、职责混用
3. 连续性元素是否集中管理
4. 哪些模板最值得优先优化
5. 是否需要后续参考资料文件夹来做更细的对标优化

本文件记录的是对当前项目提示词体系的分析结论和改进方向，不是直接照搬外部资料的实现稿。

---

## 二、已复核的提示词系统

从 `backend/core/prompt_templates.py` 看，当前提示词系统大致分为以下几类。

### 1. 全局分析

- `GLOBAL_ANALYSIS`
- 用于生成 `story_bible`
- 关注全局角色、场景、道具、风格、时间线与连续性约束

### 2. 阶段二分镜拆解

- `BATCH_DECOMPOSE`
- `MANUAL_SHOT_STRUCTURE`
- 用于把剧本拆成结构化分镜字段
- 产出 `scene / characters / props / action / camera / dialogue / emotion / duration / handoff`

### 3. 资产参考图

- `CHARACTER_SHEET`
- `SCENE_SHEET`
- `SCENE_AERIAL`
- `PROP_SHEET`

这部分主要负责视觉锚定和参考图统一。

### 4. 连续性引擎

- `HANDOFF_DECISION`
- `STAGING_DIAGRAM`
- `DIRECTOR_BOARD`
- `CONTINUITY_REVIEW`

这部分负责决定本镜怎么承接上一镜，以及是否需要站位图、导演图和复核。

### 5. 视频生成提示词

- `VIDEO_PROMPT`

当前逻辑是把静态图片提示词升级为视频提示词，再叠加镜头动态信息。

### 6. 逐镜提示词推理

- `SHOT_PROMPT_INFER`

这是最适合做“提示词瘦身”和“结构统一”的核心模板。

### 7. 分集识别

- `EPISODE_SPLIT`

用于导入时的自适应分集边界判断。

## 三、当前提示词系统的主要问题

### 1. 职责有重叠

一些信息在多个模板中重复出现，例如：

- 风格
- 场景
- 角色
- 道具
- 连续性衔接
- 运镜 / 景别
- 动作时间轴
- 台词

这会导致：

- 提示词变长
- 同一信息重复表达
- 生成目标不够纯净
- 某些无关字段被带入本次任务

### 2. 层级混用

当前模板里存在这些混用现象：

- 结构化字段和正文提示词混在一起
- 连续性判断和画面生成混在一起
- 原文上下文和可视化参数混在一起

这会让模板变得“什么都有”，但职责不够专一。

### 3. 连续性元素分散

虽然已经有连续性引擎，但相关信息仍以多种形式散落在：

- 批次任务
- 结果返回
- `story_state`
- 提示词正文

这会造成：

- 前后端显示不完全一致
- 回放和审计时要拼很多层
- 模板改动时容易影响其它模块

### 4. 任务无关字段混入

某些提示词会把与本次任务无直接关系的字段也带进去，例如：

- 过多剧情上下文
- 过多资产说明
- 过多连续性决策信息
- 过多复核信息

这会削弱提示词对当前镜头的聚焦度。

## 四、从外部参考资料中吸收的方向

参考 `docs2` 中的资料后，可以抽象出几个很明确的改进方向。

### 1. 提示词主体要更“纯”

图片提示词和视频提示词应更严格区分职责：

- 图片提示词：偏静态主体、场景、构图、风格、参考图锚定
- 视频提示词：偏首帧、运镜、动作时间轴、连续性、节奏、台词

### 2. 连续性要集中在专门模块

像以下元素，应该尽量只在连续性层出现：

- `prev_handoff`
- `decision`
- `tail_frame`
- `staging`
- `director_board`
- `camera_hint`
- `strategy`
- `source`

不要让它们散落在所有模板里。

### 3. 时间轴更重要

视频提示词尤其需要明确：

- 起势
- 过渡 / 核心动作
- 收束 / 下一段接口
- 最终姿态 / 最终站位
- 下一段如何接

### 4. 去除无关字段

提示词应该只保留“本次任务真正需要的字段”，减少过量剧情复述或无用上下文。

## 五、当前项目里最值得优先优化的模板

### 1. `SHOT_PROMPT_INFER`

这是最核心、收益最大的优化点。

原因：
- 同时面向图片和视频提示词
- 承接了结构化字段
- 可以统一决定“该保留什么、该删什么”
- 适合把主体拆分成更纯净的 image / video 两块

### 2. `VIDEO_PROMPT`

当前是 `image_prompt + dynamics` 的拼接型结构。

建议：
- 明确静态信息与动态信息的边界
- 只保留和视频相关的内容
- 去掉重复的静态描述

### 3. `HANDOFF_DECISION`

建议收紧输入，避免把不必要的上下文带入连续性判断。

### 4. `CONTINUITY_REVIEW`

建议围绕“上一镜状态 + 本镜关键图 + 本镜关键字段”来判断，不要扩大到无关叙事层。

## 六、建议的提示词结构规范

### A. 图片提示词主体建议

只保留：

- 风格
- 主体
- 场景
- 关键道具
- 构图 / 景别
- 参考图标签

### B. 视频提示词主体建议

只保留：

- 首帧锚定
- 运镜 / 景别
- 动作时间轴
- 连续性衔接
- 台词 / 旁白（若有）
- 节奏 / 时长
- 防穿帮约束

### C. 连续性判断提示词建议

只保留：

- 上一镜状态
- 本镜结构字段
- 是否长镜头
- 是否需要站位图
- 是否需要导演图
- 是否切镜头

---

## 七、建议的改进优先级

### 第一优先级

1. 重构 `SHOT_PROMPT_INFER`
2. 明确 image / video 的字段边界
3. 去掉本次任务不相关字段

### 第二优先级

1. 收紧 `VIDEO_PROMPT`
2. 收紧 `HANDOFF_DECISION`
3. 收紧 `CONTINUITY_REVIEW`

### 第三优先级

1. 统一前端展示逻辑
2. 统一导出逻辑
3. 统一审计 / 回放的快照结构

---

## 八、建议的实施策略

### 方案一：先改核心模板，再逐步收口

适合当前项目。

执行顺序建议：
1. 先做 `SHOT_PROMPT_INFER`
2. 再做 `VIDEO_PROMPT`
3. 再做连续性模板
4. 最后统一前端展示

### 方案二：先定义提示词规范，再批量改模板

优点：整体更统一。
缺点：前期耗时更长。

### 推荐方案

建议采用“先定义规范，再逐个模板改”的方式。

---

## 九、建议保留的能力

不是所有重复都要删掉。以下能力建议保留：

- `src_text` 原文回溯
- `scene_ref / action_ref / handoff_ref / dialogue_ref / characters_ref`
- `decision / strategy / source`
- `tail_frame / staging_image / director_board`
- `duration / emotion`

这些不是冗余字段，而是为了保证可追踪、可回放、可审计。

真正需要删的是“模板正文里重复堆叠这些字段”的做法，而不是删掉字段本身。

## 十、后续参考资料建议

如果后续你要继续补充参考资料，建议单独建一个文件夹，按用途分类，便于对标学习：

- `video_prompt/`：视频提示词示例
- `image_prompt/`：图片提示词示例
- `continuity/`：连续性相关案例
- `shot_structures/`：分镜结构样例
- `style_refs/`：风格与视觉基准

这样后续我可以更快地对照你的参考资料，继续做“学习理解，但不照搬”的优化。

---

## 十一、结论

当前提示词系统已经具备重构基础，但整体仍存在：

- 提示词太长
- 字段重复
- 连续性信息分散
- 图文与动态职责混用
- 部分无关字段混入本次任务

最值得先做的是：

1. 复核并重构 `SHOT_PROMPT_INFER`
2. 收紧 `VIDEO_PROMPT`
3. 统一连续性字段的使用边界
4. 清理无关任务字段

这份分析可以作为后续实际改造的起点。
